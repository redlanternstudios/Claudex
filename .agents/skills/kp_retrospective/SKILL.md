# KP Retrospective

This is analysis only unless a separate implementation task is authorized.

Read the scope, plan, final receipt, final diff, CI evidence, and `OPS/KP_SKILL_OPERATING_STANDARD.md`.

## Required analysis

1. Original plan versus actual execution
2. Authorization points used and missed
3. What worked
4. What caused friction
5. Incorrect assumptions
6. Hidden dependencies
7. Scope deviations
8. Semantic surprises
9. Gate corrections
10. Quantified results
11. Reusable lessons
12. Skill improvements
13. Next smallest action

## Required metrics

Record when available:

* First pass success
* Corrective retries
* Rework required
* Scope expansion count
* Unauthorized scope deviation
* Acceptance criteria pass rate
* Evidence completeness rate
* Gate accuracy
* Semantic surprise
* Receipt freshness
* Cycle time
* Model escalations
* Files changed
* Commands run
* External systems touched

## Score

Apply the 100 point score from `OPS/KP_SKILL_OPERATING_STANDARD.md`.

Any fabricated evidence, unauthorized production effect, hidden payment or identity semantic change, or stale final receipt caps the run at 49.

## Skill improvement rule

For every proposed skill change state:

1. Current weakness
2. Evidence from the run
3. Proposed improvement
4. Expected measurable value
5. Separate task required Yes or No

Do not edit skill files during the retrospective.
