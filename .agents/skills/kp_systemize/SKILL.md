# KP Systemize

Read `CTP_FRAMEWORK.md` and `OPS/KP_SKILL_OPERATING_STANDARD.md`.

## Interactive behavior

Guide KP through one section at a time when requested. Do not design the entire process without collecting his understanding and decision points.

## Required sections

1. Repeated problem
2. Real example
3. Trigger
4. Frequency
5. Current failure pattern
6. Desired consistent outcome
7. Verified facts
8. Assumptions
9. Unknowns
10. Upstream causes
11. Downstream consequences
12. Required judgment questions
13. KP responsibilities
14. Codex responsibilities
15. Human approval points
16. Inputs
17. Outputs
18. Evidence
19. Metrics
20. Failure behavior
21. Recovery behavior
22. Improvement loop

## Select the simplest sufficient form

1. One time task
   Use when recurrence is unlikely.
2. Checklist
   Use when the steps are known and judgment is low.
3. Fillable template
   Use when the same information must be collected.
4. Skill
   Use when Codex must guide reasoning and apply rules.
5. SOP
   Use when humans need repeatable ownership and instructions.
6. Agent
   Use when bounded autonomous work spans several steps or tools.
7. Automation
   Use when rules are predictable and human judgment is low.

Do not build an agent when a checklist solves the problem.

## Quantification design

Define:

* Input volume
* Cycle time
* First pass success
* Rework rate
* Error rate
* Evidence completeness
* Scope containment
* Outcome value
* Human intervention rate
* Cost per run
* Skill update rate
