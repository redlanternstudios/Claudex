## PRODUCT FIREBREAK GOVERNOR (Module 4)
At every session start, read /OPS/PRODUCT_FIREBREAK.md. Enforce without exception:
- ACTIVE BUILD product only: full agent team, new features allowed
- MAINTENANCE LANE products: bug fixes, compliance, critical patches only — no new features
- PARKED products: NO work unless SECURITY issues CRITICAL finding + ROBBY approves,
  OR Ro explicitly updates TODAY.md with a product reassignment,
  OR COMPLIANCE issues a legal/religious ruling requiring immediate action.

Parked product violation: dead-letter the task, post to Governance Alerts, notify ROBBY.

## MATURITY GATE (runs before ANY task creation — no exceptions)
CONCEPT ONLY: do not create tasks. Flag to ROBBY with classification reason.
PROTOTYPE: spike tasks only, max 3, no sprint assignment.
DOCUMENTED OPERATOR PLAYBOOK: full task creation permitted.
PRODUCT-READY: full sprint assignment permitted.

To reach DOCUMENTED OPERATOR PLAYBOOK, feature must have:
- Entity model (entities, fields, ownership, relationships, lifecycle)
- State model (states, transitions, entry/exit conditions, failure states)
- Control layer (what is allowed, blocked, requires review/confirmation)
- Risk classification (Low / Medium / High)

## TASK QUALITY GATE (all 6 required — no exceptions)
1. Title (imperative, specific)  2. Owner (named agent or Ro)  3. Due date (absolute)
4. Definition of Done (observable, not vague)  5. Dependencies (named explicitly)
6. Risk classification (Low / Medium / High)
Vague task = placeholder. Return to PM for hardening before creation.

- Agents that attempted parked product work (with task evidence)
