# From Rory

Status ACTIVE

Canonical capability: claudex.skill.from-rory

## Trigger

Run when KP says `FROM RORY`, `from Rory`, `Rory sent this`, `my brother sent this`, or identifies a current screenshot, message, email, file, commit, receipt, or handoff as received from Rory.

## Contract

Read `OPS/CLAUDEX_RORY_OPERATING_LOOP_CONTRACT.md` before execution.

Execute this sequence:

`Resolve artifact -> Inspect evidence -> Assign truth state -> Trace upstream -> Explain current mechanism -> Trace downstream -> Identify decision -> Name Rory action -> Define done proof -> Teach back`

Do not replace the walkthrough with a heartbeat, generic summary, or task list.

## Truth ladder

Use the strongest supported state only: `OBSERVED`, `CLAIMED`, `IN PROGRESS`, `LOCALLY VERIFIED`, `CANONICALLY VERIFIED`, or `DONE`.

A screenshot of editing, agent creation, or a success statement without authoritative readback is not `DONE`.

## Required walkthrough

1. What Rory sent and its truth state.
2. Plain language meaning.
3. Upstream objective, source, prerequisites, and owner.
4. Current mechanism and system location.
5. Downstream consumers, consequences, and unlocked work.
6. What is proven.
7. What is not yet proven.
8. KP decision or `NO DECISION`.
9. Exactly one Rory next action.
10. Exact done proof.
11. One teach back question about the system relationship.

When the artifact is visual, apply `.claude/skills/sight-engine.md` to professional articulation and legibility. Visual inspection does not prove repository, deployment, or delivery state.

Stop at explanation unless KP separately authorizes a repository change, message, external action, or state mutation.
