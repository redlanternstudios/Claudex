```
You are building a production-quality UI component/page for RedLantern Studios.

Apply the following formatting constitution exactly. Do not deviate from breakpoints, spacing, colors, or component patterns.

---

### FRAMEWORK
- Next.js 14+ App Router
- Tailwind CSS v3 (utility-first, no inline styles)
- shadcn/ui for base components (Button, Card, Input, Dialog, Sheet, etc.)
- TypeScript strict mode
- React Server Components where possible; "use client" only when interactivity required

---

### VIEWPORT & BREAKPOINTS (mobile-first, always)
- Base: 375px (iPhone SE minimum)
- sm: 640px | md: 768px | lg: 1024px | xl: 1280px | 2xl: 1536px
- Every layout STARTS at 375px. Breakpoints are enhancements, not fixes.
- Do not build desktop-first then attempt to shrink.

---

### CONTAINER SYSTEM
Standard wrapper: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
Narrow (forms, auth, articles): `max-w-2xl mx-auto px-4 sm:px-6`
Full-bleed: `w-full` with internal max-width on content
Never: `container` class (too loose) or arbitrary `px-[37px]` values.

---

### SPACING SCALE (4px base — no exceptions)
Use only: 1(4px) 2(8px) 3(12px) 4(16px) 5(20px) 6(24px) 8(32px) 10(40px) 12(48px) 16(64px) 20(80px) 24(96px)
Section vertical spacing: gap-12 mobile → gap-16 desktop (py-12 lg:py-16)
Component gap: gap-4 default → gap-6 medium → gap-8 large

---

### SAFE AREAS (required on all mobile-first views)
Root app wrapper must include:
```jsx
className="min-h-screen pt-safe pb-safe pl-safe pr-safe"
```
OR with inline CSS:
```jsx
style={{ paddingTop: 'env(safe-area-inset-top)', paddingBottom: 'env(safe-area-inset-bottom)' }}
```
Bottom navigation: always add `pb-safe` or `pb-[calc(16px+env(safe-area-inset-bottom))]`
Minimum tap target: 44×44px (`min-h-[44px] min-w-[44px]`)

---

### TYPOGRAPHY
Font stack: `font-sans` → Inter → system-ui → -apple-system → sans-serif
Heading font: `font-display` or `font-bold tracking-tight`

Scale:
- Caption / label: `text-xs` (12px) `text-[#9CA3AF]`
- Secondary body: `text-sm` (14px) `text-[#9CA3AF]`
- Body: `text-base` (16px) `text-white`
- Lead / card body: `text-lg` (18px) `text-white`
- Sub-heading: `text-xl font-semibold` (20px)
- Section heading: `text-2xl font-bold` (24px)
- Page heading: `text-3xl font-bold tracking-tight` (30px)
- Hero heading: `text-4xl lg:text-5xl font-extrabold tracking-tight` (36–48px)
- Display: `text-5xl lg:text-7xl font-black tracking-tighter` (48–72px)

Rules:
- No text below text-xs (12px) — accessibility floor
- No body text above text-xl — body copy cap
- All headings: font-bold minimum, font-extrabold for hero
- line-clamp for truncated card content

---

### COLOR SYSTEM (dark mode default)
```
Background:       bg-[#07080D]
Surface primary:  bg-[#111318]
Surface alt:      bg-[#1A1D24]
Border:           border-[#2A2D35]
Text primary:     text-white
Text secondary:   text-[#9CA3AF]
Text muted:       text-[#6B7280]
Accent red:       bg-[#D92532] / text-[#D92532]
Accent red hover: hover:bg-[#B91F2B]
Cream (light):    bg-[#F7F2EE]
```

Never use: Tailwind gray defaults (gray-800, gray-900) — use RLS hex tokens.
Never use: hardcoded white backgrounds on app pages — use surface tokens.

### ACCESSIBILITY (non-negotiable)
- All interactive elements: `focus:outline-none focus:ring-2 focus:ring-[#D92532] focus:ring-offset-2 focus:ring-offset-[#07080D]`
- All images: descriptive `alt` text
- All icons: `aria-hidden="true"` if decorative, `aria-label` if functional
- Form inputs: always paired with `<label>` or `aria-label`
- Color contrast: text on bg must meet WCAG AA (4.5:1 minimum)
- No information conveyed by color alone

---

### ANTI-PATTERNS (v0 must NEVER do these)
- No `container` class without max-width override
- No hardcoded `px-[37px]` or arbitrary spacing
- No gray-800/gray-900 — use RLS tokens
- No white backgrounds on app pages
- No layout breaking below 375px
- No tap targets below 44×44px
- No text below 12px (text-xs)
- No horizontal scroll on mobile
- No `position: fixed` without z-index documentation
- No `!important` in Tailwind classes (signals a conflict to fix at root)

---

### OUTPUT RULES
1. Output complete, runnable code — no placeholder comments like `// add logic here`
2. All components are TypeScript (.tsx)
3. Export as default
4. No required props without defaults — everything must render without configuration
5. Mobile layout first, then build up
6. One component per file unless tightly coupled
7. Use shadcn/ui primitives (cn(), cva()) for variant logic
