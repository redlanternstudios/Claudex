## CTP analysis

### Problem statement

Engineering and compliance problem: the current iOS target can compile and launch, but cannot produce a signed Release archive and does not present a safe native first screen.

### Three passes

Pass 1: Build health is strong because repository tests and the simulator compile pass. Submission still fails at signing.

Pass 2: Compilation alone hides user visible rejection risk. The native viewport is visibly broken around system safe areas, and disclosure text does not match the locked provider.

Pass 3: The real release gate is not code compilation. It is one reproducible signed archive plus an authenticated device journey whose UI, data, purchases, privacy controls, and recovery states all have receipts. That proof does not exist.

### Ten layers

1. Surface: Amina builds and launches.
2. Root cause: Native release configuration and responsive safe area behavior are incomplete.
3. First order consequence: Release archive fails and the first screen overlaps system UI.
4. Second order consequence: App Review cannot receive a valid build, and visible quality may trigger rejection.
5. Third order consequence: Shipping without a complete receipt creates repeated release uncertainty and weakens user trust.
6. Upstream dependencies: Apple team selection, provisioning, App Store record, privacy declarations, RevenueCat products, test account, and current live backend.
7. Downstream dependencies: TestFlight review, purchase testing, authenticated chat proof, account deletion proof, and final submission.
8. Failure modes: Signing failure, unsafe viewport, processor disclosure mismatch, missing privacy manifest, remote web outage, purchase failure, permission denial, and backend drift.
9. Recovery paths: Configure signing, repair safe areas and mobile navigation, align disclosures, add and validate privacy declarations, then rerun the complete matrix on simulator and physical device.
10. Strategic implication: A repeatable release lane turns each future submission into verification instead of rediscovery.
