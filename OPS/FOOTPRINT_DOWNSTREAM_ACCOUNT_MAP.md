# Footprint Downstream Account Map

Rule: Create only what Footprint needs. Do not duplicate tools.

1. Dedicated Footprint team email for approvals and ownership
2. Supabase for private records and policy state
3. n8n for controlled workflows
4. GitHub for versioned product and migration code
5. Team password manager for shared secret custody

1. Vercel for the Footprint web surface
2. Domain and DNS under By Red, LLC control
3. PostHog for minimum useful product measurement
4. Sentry for production error monitoring

1. Resend or the existing approved RedLantern email provider
2. Domain authentication records
3. Reply and opt out handling
4. Dedicated Footprint approval inbox

2. Homira as approved public Founding Steward
3. Ro as managing authority
4. Keymon as backup reviewer and stop authority
5. No personal password or browser session sharing
