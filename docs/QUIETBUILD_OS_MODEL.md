# QUIETBUILD OS — the model

Version 1.0 · 2026-07-02 · The umbrella. Names the hierarchy so the engines stop feeling like parallel brands.

QuietBuild OS is RedLantern's proprietary AI-native delivery system: one founder directing AI engines over an agent execution layer, building a product portfolio from one shared brain.

```text
QuietBuild OS
  └── Claudex          = canonical brain, bridge, governance, truth, receipts
        ├── Bridge         = current cross-engine state (OPS/BRIDGE.json)
        ├── TruthSerum     = verification doctrine (claim → evidence → receipt)
        ├── SafetyEngine   = risk gate (pre-execution)
        ├── BrainSmart     = deliberation gate (high blast radius only)
        ├── SilentEngine   = model routing + quiet reasoning policy
        ├── Robby PA       = conductor / operator (never self-approves)
        ├── SwarmClaw      = agent execution org
        └── Delivery Kernel = repeatable shipping playbook
  QBos = source inventory / legacy engine warehouse (not the production OS)
```

## Naming decisions this locks
- **Claudex** is the canonical governance brain. Doctrine lives here.
- **`QuietBuild-OS` the repo** must be reconciled: either it becomes the public front for this model or it defers to Claudex. Do not leave both claiming the title. (Open decision — see `REPO_INVENTORY.md`.)
- The engines are **not peers**: SafetyEngine and BrainSmart are components/gates; SwarmClaw is an org; TruthSerum/SilentEngine are doctrine-backed engines. See `ENGINE_REGISTRY.md` for maturity.

## Why this is the moat (and why it isn't yet)
The moat is **proven, tested, integrated engines that measurably ship faster and more truthfully** than a solo founder with an IDE. Documentation is not the moat — anyone can generate docs. Today the model is real in architecture and thin in proof. It becomes a moat when one product loop is proven end-to-end through these gates with real receipts.

## The one thing that converts this from claim to moat
Prove a loop (Amina) through: SafetyEngine gate → build → TruthSerum verify → receipt → GREEN. That single pass validates the model and generates the first real evidence the whole hierarchy is more than naming.
